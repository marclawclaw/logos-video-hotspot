# Demo Files

This folder contains recordings and mockups of the Video Hotspot application.
The main README embeds all of these — this file explains how to replay them locally.

---

## CLI Demo

| File | Format | Description |
|------|--------|-------------|
| `cli-demo.cast` | asciinema v2 | Raw terminal recording (replayable) |
| `cli-demo.gif`  | Animated GIF | Rendered for GitHub embedding |

### Replay the cast (live terminal, with timing)

```bash
asciinema play demo/cli-demo.cast
```

Install asciinema if needed:

```bash
pip install asciinema
# or: brew install asciinema
```

### Re-record

```bash
asciinema rec demo/cli-demo.cast \
  --command "bash demo/record-cli.sh" \
  --title "Video Hotspot CLI Demo" \
  --cols 88 --rows 35 \
  --overwrite
```

Then regenerate the GIF:

```bash
agg demo/cli-demo.cast demo/cli-demo.gif --theme monokai --font-size 13 --speed 1.5
```

`agg` binaries: <https://github.com/asciinema/agg/releases>

## GUI Demo Video (`gui-demo.mp4`)

| Attribute | Value |
|-----------|-------|
| Duration  | ~2m 25s |
| Resolution | 1280×800 |
| Video | H.264, 25 fps |
| Audio | AAC 128 kbps narration |
| Tabs shown | Map · Upload · Downloads · Settings |

### Timed segments

| Timestamp | Tab | Voiceover highlights |
|-----------|-----|---------------------|
| `00:00–00:36` | **Map** | Geo pins, timeline slider filtering |
| `00:36–01:27` | **Upload** | File queue, progress bars, DUPLICATE badge |
| `01:27–01:39` | **Downloads** | Storage bar, owned videos |
| `01:39–01:57` | **Settings** | Storage limit, auto-monitoring, Logos node |
| `01:57–end`   | **Map** | Closing stack context (Waku · Codex · Nomos) |

### Regenerate

```bash
bash demo/gen-gui-video.sh
```

Requires: `python3`, `rsvg-convert` (from `librsvg2-bin`), `ffmpeg`, `espeak-ng`

```bash
# Install dependencies (Debian/Ubuntu/Raspberry Pi OS)
sudo apt-get install -y librsvg2-bin ffmpeg espeak-ng
```

The script runs the full pipeline:
1. **`gen-gui-mockup.py`** → 4 SVG frames (Map, Upload, Downloads, Settings)
2. **`rsvg-convert`** → 4 PNG screenshots at 1280×800
3. **`ffmpeg`** → timed video segments per tab, concat'd
4. **`narration.wav`** → muxed as AAC audio track

### Why screenshots instead of live Qt recording

Qt6/QML requires hardware OpenGL (or Mesa software rasterisation) to render
`QQuickWidget`. Under Xvfb without a GPU, the window renders as a white/blank
frame — `x11grab` captures pure white regardless of what the QML defines.

The screenshot approach guarantees content because we control every pixel via the
Python SVG generator, bypassing the Qt renderer entirely. The output frames
are pixel-perfect representations of `VideoHotspotApp.qml` (same colours, same
layout, same content) — just rendered with `librsvg` instead of Qt.

This approach works on any headless server, CI runner, or Raspberry Pi without
a GPU.

## Demo Script

`record-cli.sh` drives the CLI demo session. It wipes the local DB before
running so every recording starts from a clean state.

```bash
bash demo/record-cli.sh   # dry run (no recording)
```
